Real-Time World Index: Technical Overview

A concise look at RTWI architecture, data flow, and query primitives.

Real-Time World Index (RTWI) is a distributed sensing and indexing system designed to represent physical-world change as a live, queryable stream. Edge devices capture multimodal observations continuously, attach trusted time and location metadata, and stream events into a normalization layer. The result is not a static map artifact but a rolling state model of what is happening across space and time.

At ingestion, RTWI validates payload structure, deduplicates near-identical events, and binds records to H3 cells to create spatially stable addressability. H3 res-9 provides a practical balance between locality and scale, allowing downstream systems to aggregate and drill down without custom geospatial infrastructure per application. Storage is optimized for temporal access patterns so recency-first queries remain fast under continuous write load.

On top of this index, world.query() exposes a simple API contract for developers and operators. Instead of pulling raw files, clients request world-state primitives by cell, interval, and modality. This query layer is built for low-latency operational use cases and agent workflows where fresh physical context is required to decide, route, and coordinate in the real world.
